# 手動テスト項目表 — v1.0.3 新機能

## 前提条件

- Ren'Py SDK 8.x がインストール済みで `renpyCode.sdkPath` を設定済み
- the_question プロジェクト（または `game/` ディレクトリを持つ任意の Ren'Py プロジェクト）を開いている
- 拡張機能がビルド・ロード済み（F5 での開発実行 or .vsix インストール）
- ATL プレビューとセーブインスペクターのテストには Pro ライセンスが有効であること

---

## 1. 画像ホバープレビュー（Free）

### 1.1 scene 文で一致する画像がある場合
1. `game/script.rpy` を開く
2. `scene bg uni` のような行を見つける（または `scene bg room` を追加）
3. 対応する画像ファイルが存在することを確認（`game/images/bg room.png`、`game/images/bg/room.png`、`game/images/bg_room.png` のいずれか）
4. scene 文の `bg` や `room` にマウスカーソルを合わせる
5. **期待結果**: ホバーツールチップが表示される：
   - 「**画像** `bg room`」ヘッダー
   - サムネイル画像（幅約300px）
   - `game/images/bg/room.png` のようなファイルパス

### 1.2 show 文でキャラクター画像
1. `show eileen happy` のような行を見つける
2. `game/images/eileen happy.png` または `game/images/eileen/happy.png` が存在することを確認
3. show 文の `eileen` にホバー
4. **期待結果**: キャラクター画像のサムネイルがホバーに表示される

### 1.3 show screen は対象外
1. `show screen preferences` を見つけるか追加する
2. `preferences` にホバー
3. **期待結果**: 画像プレビューは表示されない。screen が定義されていれば screen のホバーが表示される

### 1.4 一致する画像ファイルがない場合
1. `scene bg nonexistent_place` を追加
2. `bg` や `nonexistent_place` にホバー
3. **期待結果**: 画像名のテキストのみ表示、サムネイルなし、エラーなし

### 1.5 アンダースコア命名規則
1. `game/images/bg_park.png` に画像を配置
2. スクリプトに `scene bg park` を追加
3. 画像名にホバー
4. **期待結果**: アンダースコア変換で画像が解決され、サムネイル表示

### 1.6 サブディレクトリ命名規則
1. `game/images/bg/park.png` に画像を配置
2. スクリプトに `scene bg park` を追加
3. 画像名にホバー
4. **期待結果**: サブディレクトリ構造で画像が解決され、サムネイル表示

### 1.7 image 定義文でのホバー
1. `image bg test = "images/bg/test.png"` を追加し、ファイルも配置
2. 定義文の `bg` にホバー
3. **期待結果**: 定義情報とともにサムネイルが表示される

## 2. スクリーン定義ツリー（Free）

### 2.1 サイドバーにツリーが表示される
1. アクティビティバー（左サイドバー）の RenPy Code アイコンをクリック
2. 「スクリーン」セクションを探す（ダッシュボードと変数の間）
3. **期待結果**: プロジェクト内の全 screen 定義が一覧表示される（例: `screen say(who, what)`、`screen main_menu` 等）

### 2.2 ウィジェット階層
1. screen ノードを展開する（例: `screen say(who, what)`）
2. **期待結果**: 子ウィジェットがツリー形式で表示される（例: `vbox`、`text`、`hbox`、`textbutton`）
3. ネストしたウィジェットは親の子として表示される

### 2.3 クリックで該当行に移動
1. スクリーンツリーの任意のノードをクリック
2. **期待結果**: エディタが .rpy ファイルの該当行にジャンプする

### 2.4 編集後の自動更新
1. `game/screens.rpy` を開く
2. 既存の screen 内にウィジェットを追加（例: vbox の中に `text "test"` を追加）
3. ファイルを保存
4. **期待結果**: スクリーンツリーが更新され、新しいウィジェットが表示される

### 2.5 ウィジェットアイコン
1. 様々なウィジェットを含む screen を展開
2. **期待結果**: ウィジェットの種類に応じたアイコンが表示される：
   - `vbox`/`hbox` → レイアウトアイコン
   - `button`/`textbutton` → 再生アイコン
   - `text` → 文字列アイコン
   - `add` → 追加アイコン
   - `if`/`elif`/`else` → 比較アイコン
   - `use` → 参照アイコン

### 2.6 screen 定義がないプロジェクト
1. screen 定義のないプロジェクトを開く
2. **期待結果**: スクリーンセクションが空、エラーなし

## 3. ドキュメントフォーマッター（Free）

### 3.1 タブ→スペース変換
1. タブインデントの .rpy ファイルを作成または開く：
   ```
   label start:
   	"Hello"
   	jump end
   ```
2. `Shift+Alt+F`（または右クリック → ドキュメントのフォーマット）
3. **期待結果**: タブが4スペースに変換される：
   ```
   label start:
       "Hello"
       jump end
   ```

### 3.2 末尾スペース削除
1. 行末にスペースを追加: `label start:   `（末尾に3スペース）
2. フォーマットを実行
3. **期待結果**: 末尾スペースが除去: `label start:`

### 3.3 define/default の = 前後のスペース
1. `define e=Character("Eileen")` を追加
2. フォーマット
3. **期待結果**: `define e = Character("Eileen")`
4. 追加確認: `default book  =  False` → `default book = False`

### 3.4 ブロックヘッダーのコロン前のスペース
1. `label start :` を追加
2. フォーマット
3. **期待結果**: `label start:`
4. 追加確認: `screen main_menu :`、`if x > 0 :`、`menu :` も同様

### 3.5 キーワード後の複数スペース
1. `jump  start`（スペース2つ）を追加
2. フォーマット
3. **期待結果**: `jump start`（スペース1つ）
4. 追加確認: `scene  bg room`、`show  eileen` も同様

### 3.6 連続空行の圧縮
1. 2つの label ブロックの間に5行以上の空行を挿入
2. フォーマット
3. **期待結果**: 連続空行が最大2行に圧縮される

### 3.7 スペースのみの行
1. スペースだけの行を追加（例: 4スペース、テキストなし）
2. フォーマット
3. **期待結果**: 行が完全に空になる（0文字）

### 3.8 整形済みファイルで変更なし
1. 既に整形されている .rpy ファイルを開く
2. フォーマット
3. **期待結果**: 変更が発生しない（差分なし）

## 4. ATL アニメーションプレビュー（Pro）

### 4.1 コマンドパレットからのアクセス
1. コマンドパレットを開く（`Ctrl+Shift+P`）
2. 「ATL」または「Preview ATL」と入力
3. **期待結果**: 「RenPy Code: ATLアニメーションをプレビュー」コマンドが表示される

### 4.2 transform の選択
1. プロジェクトに少なくとも1つの `transform` 定義があることを確認（the_question の `screens.rpy` にあるはず）
2. コマンドを実行
3. **期待結果**: プロジェクト内の全 transform の一覧が表示され、名前・パラメータ・ファイル位置が確認できる

### 4.3 アニメーション表示
1. 一覧から transform を選択
2. **期待結果**: エディタ横に WebView パネルが開き、以下が表示される：
   - transform 名とパラメータのヘッダー
   - 再生時間、リピート状態、キーフレーム数
   - ATL プロパティに基づいてアニメーションする色付きの矩形
   - 時間とプロパティのキーフレームテーブル

### 4.4 Replay ボタン
1. WebView の「Replay」をクリック
2. **期待結果**: アニメーションが最初から再生される

### 4.5 Pause/Resume ボタン
1. アニメーション中に「Pause / Resume」をクリック
2. **期待結果**: アニメーションが一時停止する。もう一度クリックで再開

### 4.6 カスタム transform のテスト
1. .rpy ファイルに以下を追加:
   ```
   transform my_spin:
       rotate 0
       2.0
       rotate 360
       repeat
   ```
2. プレビューコマンドを実行し `my_spin` を選択
3. **期待結果**: 矩形が2秒かけて360度回転し、繰り返される

### 4.7 ライセンスチェック
1. Pro ライセンスを無効化（設定の `renpyCode.license.key` をクリア）
2. コマンドを実行
3. **期待結果**: ライセンス有効化ダイアログが表示される
4. ライセンスキーを再入力して復旧

### 4.8 transform が存在しない場合
1. transform 定義のないプロジェクトを開く
2. コマンドを実行
3. **期待結果**: 「プロジェクト内にトランスフォームが見つかりません」メッセージ

## 5. セーブデータインスペクター（Pro）

### 5.1 コマンドパレットからのアクセス
1. コマンドパレットを開く（`Ctrl+Shift+P`）
2. 「Save」または「Inspect」と入力
3. **期待結果**: 「RenPy Code: セーブファイルを検査」コマンドが表示される

### 5.2 セーブファイルの選択
1. `game/saves/` に少なくとも1つの .save ファイルがあることを確認（ゲームを少しプレイしてセーブする）
2. コマンドを実行
3. **期待結果**: 全 .save ファイルの一覧がファイルサイズ付きで表示される

### 5.3 メタデータ表示
1. セーブファイルを選択
2. **期待結果**: WebView パネルが開き、以下が表示される：
   - ファイル名（ヘッダー）
   - セーブ名（オートセーブの場合は「(auto)」）
   - ゲームバージョン
   - Ren'Py バージョン
   - プレイ時間（Xh Xm Xs 形式）
   - 作成日時

### 5.4 スクリーンショット表示
1. セーブファイルを選択
2. **期待結果**: パネル上部にセーブ時のスクリーンショットが表示される

### 5.5 ゲーム変数の表示
1. 「Game Variables」セクションまでスクロール
2. **期待結果**: セーブデータ内の変数がテーブル形式で表示される：
   - 変数名が青いモノスペースフォント
   - 真偽値はオレンジ
   - 数値は緑
   - 文字列は黄色
   - オブジェクトはグレー
3. ユーザー定義変数が含まれる（例: the_question の `book`）
4. `_` で始まる内部変数は除外されている

### 5.6 変数フィルター
1. フィルター入力欄に文字を入力（例: 「book」）
2. **期待結果**: テーブル行がリアルタイムでフィルタされ、一致する変数のみ表示される
3. フィルターをクリア
4. **期待結果**: 全変数が再表示される

### 5.7 アーカイブ内容
1. 「Archive Contents」セクションまでスクロール
2. **期待結果**: ZIP エントリがサイズ付きで一覧表示される：
   - `screenshot.png (xxx KB)`
   - `json (xxx B)`
   - `log (xxx KB)`
   - `renpy_version (xx B)`
   - など

### 5.8 saves ディレクトリがない場合
1. `game/saves/` が存在しないプロジェクトを開く
2. コマンドを実行
3. **期待結果**: 「セーブディレクトリが見つかりません」メッセージ

### 5.9 セーブファイルが0件の場合
1. `game/saves/` は存在するが中身が空
2. コマンドを実行
3. **期待結果**: 「セーブファイルが見つかりません」メッセージ

### 5.10 ライセンスチェック
1. Pro ライセンスを無効化
2. コマンドを実行
3. **期待結果**: ライセンス有効化ダイアログが表示される
