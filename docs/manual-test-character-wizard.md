# 手動テスト項目表 — キャラクターウィザード

## 前提条件

- the_question プロジェクト（または `define` でキャラクターが定義された Ren'Py プロジェクト）を開いている
- Pro ライセンスが有効
- コマンドパレットから「RenPy Code: Character Wizard」を実行してウィザードを開く

### テスト用サンプルスクリプト

以下を `game/script.rpy` 等に記述しておくとテストしやすい:

```renpy
define e = Character("Eileen", color="#c8ffc8", image="eileen")
define s = Character("Sylvie", color="#c8c8ff")
define m = Character("Me", kind=nvl_character)
```

`game/images/` フォルダに以下のようなファイルがあるとよい:

```
images/eileen happy.png
images/eileen concerned.png
images/sylvie smile.png
```

---

## 1. 起動とライセンス

### 1.1 Pro ライセンスなしでの実行
1. ライセンスキーを未設定にする
2. コマンドパレットから「RenPy Code: Character Wizard」を実行
3. **期待結果**: Pro 機能アンロックのダイアログが表示され、ウィザードは開かない

### 1.2 Pro ライセンスありでの実行
1. 有効なライセンスキーを設定する
2. コマンドパレットから「RenPy Code: Character Wizard」を実行
3. **期待結果**: キャラクターウィザードパネルが開く

### 1.3 ダッシュボードからの起動
1. サイドバーの RenPy Code ダッシュボードを開く
2. 「Pro Tools」セクションの「👤 Character Wizard」ボタンをクリック
3. **期待結果**: キャラクターウィザードパネルが開く（コマンドパレットと同じ動作）

### 1.4 インデックスが空の場合
1. プロジェクトを開いた直後（インデックス未完了時）に実行
2. **期待結果**: 自動的にワークスペースがインデックスされてからウィザードが開く

---

## 2. キャラクター一覧（カードグリッド）

### 2.1 初期表示
1. キャラクターが定義されたプロジェクトでウィザードを開く
2. **期待結果**:
   - 「キャラクターウィザード」タイトルが表示される
   - 「+ 新規キャラクター」ボタンが表示される
   - 右側にキャラクター数（例: 「3 characters」）が表示される
   - 各キャラクターがカード形式でグリッド表示される

### 2.2 カードの内容
1. 各カードの表示を確認
2. **期待結果**:
   - 色付きアバター（キャラクターの color に基づく丸アイコン、名前の頭文字入り）
   - キャラクター表示名（例: "Eileen"）
   - 変数名（例: `e`）がモノスペースで表示
   - color が定義されていればカラースウォッチ + hex 値
   - image タグが定義されていれば表示（例: `img: eileen`）
   - 表情数（例: 「3 expressions」）

### 2.3 カードのソート順
1. 複数キャラクターの並び順を確認
2. **期待結果**: 変数名のアルファベット順でソートされている

### 2.4 色なしキャラクター
1. `define n = Character("Narrator")` のように color 未指定のキャラクターを確認
2. **期待結果**: アバターがデフォルト色（グレー）で表示される

---

## 3. キャラクター詳細パネル

### 3.1 カードクリックで詳細表示
1. キャラクターカードをクリック
2. **期待結果**:
   - クリックしたカードに選択枠がつく
   - 詳細パネルが表示される
   - タイトルに「表示名 (変数名)」が表示される

### 3.2 プロパティ一覧
1. 詳細パネルのプロパティ欄を確認
2. **期待結果**: 以下のプロパティが該当するものだけ表示される
   - Variable（変数名）
   - Display Name（表示名）
   - Color（色 — カラースウォッチ付き）
   - Image Tag（画像タグ）
   - who_color（カラースウォッチ付き）
   - what_color（カラースウォッチ付き）
   - callback
   - File（ファイル名:行番号）

### 3.3 詳細パネルを閉じる
1. 詳細パネル右上の「✕」ボタンをクリック
2. **期待結果**: 詳細パネルが非表示になり、カードの選択枠も解除される

### 3.4 別のキャラクターを選択
1. キャラクター A を選択中に、キャラクター B のカードをクリック
2. **期待結果**: 選択が B に切り替わり、詳細パネルの内容が更新される

   - サムネイル画像が表示される
   - ソースが「file」と表示される

## 6. 新規キャラクター作成

### 6.1 フォーム表示
1. 「+ 新規キャラクター」ボタンをクリック
2. **期待結果**: 作成フォームが表示される（変数名、表示名、色、画像タグのフィールド）

### 6.2 フォームの閉じ方
1. フォーム内の「Cancel」ボタンをクリック
2. **期待結果**: フォームが非表示になる

### 6.3 カラーピッカーの連動
1. カラーピッカー（色選択ボックス）で色を変更
2. **期待結果**: テキストフィールドに hex 値（例: `#ff0000`）が反映される
3. テキストフィールドに `#00ff00` と入力
4. **期待結果**: カラーピッカーの色が更新される

### 6.4 プレビュー
1. 変数名に `t`、表示名に `Taro`、色に `#ffcc00` を入力
2. 「Preview」ボタンをクリック
3. **期待結果**: 以下のようなコードがプレビュー表示される
   ```
   define t = Character("Taro", color="#ffcc00")
   ```

### 6.5 画像タグ付きプレビュー
1. 画像タグフィールドに `taro` を入力して「Preview」
2. **期待結果**:
   ```
   define t = Character("Taro", color="#ffcc00", image="taro")
   ```

### 6.6 バリデーション（未入力）
1. 変数名または表示名を空のまま「Insert Definition」をクリック
2. **期待結果**: プレビュー欄に「変数名と表示名を入力してください。」と表示される

### 6.7 既存ファイルへの挿入
1. 既にキャラクターが定義されたプロジェクトで新規キャラクターを作成
2. 変数名: `k`, 表示名: `Ken`, 色: `#ff8888` を入力
3. 「Insert Definition」をクリック
4. **期待結果**:
   - 既存のキャラクター定義があるファイルに `define k = Character("Ken", color="#ff8888")` が挿入される
   - 挿入位置は最後のキャラクター定義の直後
   - テキストエディタが開き、挿入行がハイライトされる
   - 通知メッセージが表示される
   - ウィザードが自動更新され、新キャラクターがカード一覧に追加される

### 6.8 definitions.rpy の自動作成
1. キャラクター定義が一切ないプロジェクトで新規キャラクターを作成
2. `game/definitions.rpy` も `game/characters.rpy` も `game/script.rpy` も存在しない状態
3. **期待結果**:
   - `game/definitions.rpy` が自動作成される
   - ファイルにヘッダコメントと define 文が書き込まれる
   - テキストエディタで開かれる

### 6.9 script.rpy へのフォールバック
1. キャラクター定義はないが `game/script.rpy` が存在するプロジェクトで新規作成
2. **期待結果**: `game/script.rpy` に define 文が挿入される

## 8. エッジケース

### 8.1 キャラクター0件のプロジェクト
1. `define ... = Character(...)` が一切ないプロジェクトでウィザードを開く
2. **期待結果**:
   - 空のグリッド
   - 「キャラクターがまだ定義されていません...」のガイダンスメッセージが表示される
   - 「+ 新規キャラクター」ボタンは使用可能

### 8.2 Character 以外の define
1. `define config.name = "My Game"` のような非キャラクター定義がある場合
2. **期待結果**: Character() 以外の define は表示されない

### 8.3 複雑な Character 定義
1. 以下のような複雑な定義をテスト:
   ```renpy
   define e = Character("Eileen", color="#c8ffc8", image="eileen", who_color="#c8ffc8", what_color="#ffffff", callback=my_callback)
   ```
2. **期待結果**: color, image, who_color, what_color, callback がすべて正しくパースされ表示される

### 8.4 翻訳可能な名前
1. `define e = Character(_("Eileen"))` のように `_()` で囲まれた名前
2. **期待結果**: 表示名が "Eileen" として正しく抽出される

### 8.5 パネルの再表示
1. ウィザードのタブを閉じて → 再度コマンドを実行
2. **期待結果**: 新しいパネルが開き、最新のキャラクター情報が表示される

### 8.6 パネルを隠して再表示
1. ウィザードのタブを他のタブの裏に隠す → 再度タブをクリック
2. **期待結果**: `retainContextWhenHidden: true` により状態が保持されている

### 8.7 .rpy 編集後のリフレッシュ
1. ウィザードを開いたまま、テキストエディタでキャラクター定義を追加・変更
2. ウィザードのコマンドを再実行
3. **期待結果**: ウィザードが更新され、変更が反映される

### 8.8 日本語キャラクター名
1. `define t = Character("太郎", color="#ffcc00")` のような日本語名
2. **期待結果**: 表示名「太郎」、アバターの頭文字が「太」になる
