# 手動テスト項目表 — ノードエディタ

## 前提条件

- the_question プロジェクト（または複数ラベルを持つ Ren'Py プロジェクト）を開いている
- Pro ライセンスが有効
- コマンドパレットから「RenPy Code: Show Story Flow Graph」を実行してエディタを開く

---

## 1. 表示・描画

### 1.1 初期表示
1. コマンドを実行
2. **期待結果**: ノードエディタが開き、プロジェクト内の全ラベルがノードとして表示される
3. ノード間の jump/call 関係がベジェ曲線で接続されている
4. 自動レイアウトで整列されている

### 1.2 ノードの種類による色分け
1. グラフ内の各ノードを確認
2. **期待結果**:
   - `start` ラベル → 緑
   - 通常ラベル → 青
   - menu を含むラベル → オレンジ
   - 行き止まり（jump/return なし）→ 赤
   - 孤立ラベル（参照されていない）→ グレー

### 1.3 ノード情報の表示
1. 各ノードの中身を確認
2. **期待結果**:
   - ラベル名がタイトル表示
   - ダイアログ行数、menu/return の有無がサブタイトル表示
   - 最初のダイアログがプレビュー表示

### 1.4 エッジの種類
1. 各エッジを確認
2. **期待結果**:
   - jump → 青い実線 + 矢印
   - call → 紫の破線 + 矢印
   - choice → オレンジの実線 + 矢印

### 1.5 統計情報
1. 画面左下を確認
2. **期待結果**: 「X labels · Y connections」と表示されている

---

## 2. パン・ズーム操作

### 2.1 マウスホイールでズーム
1. マウスホイールを上下にスクロール
2. **期待結果**: カーソル位置を中心にズームイン/アウトする
3. ズームレベル表示（ツールバー）が更新される

### 2.2 ドラッグでパン
1. キャンバスの空白部分をマウスドラッグ
2. **期待結果**: グラフ全体がスクロールする
3. カーソルが grab → grabbing に変わる

### 2.3 ツールバーのズームボタン
1. 「+」ボタンをクリック
2. **期待結果**: 画面中央を基準にズームイン
3. 「−」ボタンでズームアウト

### 2.4 Fit to View
1. ツールバーの ⛶ ボタンをクリック
2. **期待結果**: 全ノードが画面内に収まるようにズーム・パンが調整される

### 2.5 ズーム範囲制限
1. ホイールで極端にズームイン/アウト
2. **期待結果**: 10%〜500% の範囲で制限される

---

## 3. ミニマップ

### 3.1 ミニマップ表示
1. 画面右下を確認
2. **期待結果**: 全ノードが縮小表示され、ノードの色分けが反映されている

### 3.2 ビューポート表示
1. ミニマップ内の半透明の青い矩形を確認
2. **期待結果**: 現在の表示範囲を示している

### 3.3 パン・ズームとの連動
1. メインキャンバスをパン/ズームする
2. **期待結果**: ミニマップのビューポート矩形がリアルタイムで追従する

---

## 4. ノード選択・移動

### 4.1 ノードクリックで選択
1. ノードをクリック
2. **期待結果**: ノードに青い選択枠が表示される

### 4.2 空白クリックで選択解除
1. キャンバスの空白部分をクリック
2. **期待結果**: 全ノードの選択が解除される

### 4.3 Shift+クリックで複数選択
1. ノード A をクリック → Shift を押しながらノード B をクリック
2. **期待結果**: A と B 両方に選択枠が表示される

### 4.4 ノードのドラッグ移動
1. ノードをドラッグ
2. **期待結果**: ノードが移動し、接続エッジも追従する

### 4.5 複数ノードの同時移動
1. Shift+クリックで複数ノードを選択 → ドラッグ
2. **期待結果**: 選択された全ノードが同時に移動する

---

## 5. エッジ接続（コード生成）

### 5.1 ポートからドラッグでエッジ作成
1. ノード下部の出力ポート（丸）からドラッグを開始
2. 別のノード上部の入力ポートまでドラッグしてリリース
3. **期待結果**:
   - 新しいエッジが作成される
   - .rpy ファイルに `jump <target>` が自動挿入される
   - ソースファイルを開いて挿入を確認

### 5.2 ドラッグ中のプレビュー
1. 出力ポートからドラッグを開始し、まだリリースしない
2. **期待結果**: 破線のプレビューエッジがカーソルに追従する
3. カーソルが十字になる

### 5.3 自分自身への接続防止
1. ノードの出力ポートから同じノードの入力ポートにドラッグ
2. **期待結果**: エッジが作成されない

### 5.4 コンテキストメニューからの接続
1. ノードを右クリック → 「Add Jump from here...」
2. ターゲットラベル名を入力
3. **期待結果**: エッジが作成され、.rpy に jump が挿入される

### 5.5 call 接続
1. ノードを右クリック → 「Add Call from here...」 → ターゲット入力
2. **期待結果**: 破線の call エッジが作成され、.rpy に `call <target>` が挿入される

---

## 6. ノード作成（コード生成）

### 6.1 ツールバーからラベル追加
1. ツールバーの「+ Label」ボタンをクリック
2. ダイアログにラベル名（例: `my_new_scene`）を入力して OK
3. **期待結果**:
   - 新しいノードがグラフに追加される
   - .rpy ファイルに以下が自動生成される：
     ```
     label my_new_scene:
         pass
     ```

### 6.2 メニュー追加
1. ノードを選択
2. ツールバーの「+ Menu」ボタンをクリック
3. 選択肢を入力（改行区切り）して OK
4. **期待結果**:
   - 選択したノードに menu が追加される
   - .rpy に以下が自動生成される：
     ```
         menu:
             "Choice 1":
                 pass
             "Choice 2":
                 pass
     ```
   - ノードのタイプがオレンジ（choice）に変わる

### 6.3 コンテキストメニューからメニュー追加
1. ノードを右クリック → 「Add Menu...」
2. **期待結果**: デフォルト2選択肢の menu が .rpy に挿入される

---

## 11. 双方向同期

### 11.1 .rpy 編集 → グラフ更新
1. ノードエディタを開いたまま、テキストエディタで .rpy を開く
2. 新しいラベルを追加して保存
3. **期待結果**: ノードエディタに新しいノードが自動的に追加される

### 11.2 jump 追加 → エッジ更新
1. テキストエディタで既存ラベル内に `jump <target>` を追加して保存
2. **期待結果**: ノードエディタに新しいエッジが表示される

### 11.3 ラベル削除 → ノード消失
1. テキストエディタでラベルブロック全体を削除して保存
2. **期待結果**: ノードエディタから対応ノードが消える

### 11.4 グラフ編集 → .rpy 更新
1. ノードエディタでエッジを追加（ポートドラッグ）
2. テキストエディタで .rpy を確認
3. **期待結果**: `jump <target>` が挿入されている
